Проект по сетевым технологиям. Передача текстового файла с использованием WCF.

Проект является программой, состоящей из двух частей - клиентской и серверной. Клиентская часть посылает на серверную адрес файла. Серверная часть - служба WCF, запущенная на IIS сервере, принимает запрос и пытается считать файл по полученному адресу. Затем результат(текст или данные об ошибке) посылается обратно и отображается в основном окне клиента.

Основное окно клиента.

На изображении видно, что большую часть основного окна клиента занимает большое прямоугольное поле, в котором после скачивания будет отображен текст файла.Под ним находится поле для ввода адреса файла. Справа от них расположены пять кнопок:"Загрузить","Открыть","Настройка", кнопка на которой изображен красный крестик и "Default". Кнопка "Загрузить" посылает адрес из адресного поля для загрузки файла. Кнопка "Открыть" открывает диалоговое окна выбора файла для открытия на локальном компьютере. Кнопка с красным крестом очищает оба текстовых поля, а кнопка "Default" меняет адрес файла на значение по умолчанию. При нажатии кнопки "Настройка" открывается вспомогательное окно, в котором можно изменить адрес службы.

Окно настроек.

Окно настроек содержит текстовое поле с текущим адресом WCF службы, который можно изменить. Под ним находятся три кнопки: "Тест службы","ОК","По умолчанию". Кнопка "Тест службы" посылает тестовый запрос на указанный в текстовом поле адрес, кнопка "ОК" сохраняет изменения, кнопка "По умолчанию" меняет адрес WCF службы на адрес по умолчанию.
